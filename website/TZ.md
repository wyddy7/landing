# Техническое задание: Лендинг на GitHub Pages

## Концепция

**Цель**: Создать минималистичный лендинг-портфолио с единственным интерактивным элементом на странице, оптимизированный для чтения LLM и поисковых систем.

## Требования

### 1. Технический стек
- ✅ **TypeScript** - типизация
- ✅ **React 18** - UI библиотека
- ✅ **Vite** - сборщик (быстрая сборка для GitHub Pages)
- ✅ **Tailwind CSS** - стилизация
- ✅ **Framer Motion** - анимации
- ✅ **shadcn/ui структура** - организация компонентов

### 2. Дизайн и UX

#### Цветовая схема
- **Фон**: Черный (#000000)
- **Текст**: Белый (#FFFFFF)
- **Акценты**: Белые на черном
- **Контраст**: >4.5:1 (WCAG AA)

#### Единственный элемент
- Компонент `morphing-card-stack` - единственный интерактивный элемент на странице
- Три режима отображения: stack, grid, list
- Swipe-навигация для мобильных устройств

### 3. SEO и LLM-оптимизация

#### Мета-теги
- Title: "AI Systems Engineer | LLM Orchestration | wyddy7"
- Description: 150 символов с ключевыми словами
- Keywords: AI backend, Python, LLM orchestration, etc.

#### Структура контента
- Семантические заголовки (H1-H3)
- Кластеры запросов для LLM-поиска
- Schema.org JSON-LD (Person для резюме)
- PAA-like блоки (FAQ)

#### Читаемость для LLM
- Чистая HTML-структура
- Семантические теги
- Минимум JavaScript для критического контента
- Структурированные данные

### 4. Функциональность

#### Компонент карточек
- Отображение проектов/навыков
- Переключение режимов (stack/grid/list)
- Swipe-навигация
- Клик для расширения карточки

#### Данные карточек
- Название проекта/навыка
- Описание
- Иконка (lucide-react)
- Опциональный цвет фона

### 5. Деплой

#### GitHub Pages
- Автоматический деплой через GitHub Actions
- База пути: `/landing/` (или имя репозитория)
- Сборка в папку `dist/`

#### Yandex.Metrica
- Скрипт в `<head>`
- Трекинг кликов, скролла, тем
